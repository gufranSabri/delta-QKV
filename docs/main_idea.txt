TASK: Implement activation-tensor feature extraction for LLM hallucination detection.

CONTEXT
For a given LLM and a given input (prompt + generated response), we extract 
per-token, per-layer Q, K, V vectors. For each of Q, K, V separately, we build 
a tensor of shape:

    (T, L, D, 3)

where:
  T = number of tokens in the response
  L = number of transformer layers
  D = d_model (hidden dimension; if working per-head instead of concatenated 
      heads, clarify with me before assuming — default to concatenated heads, 
      i.e. D = d_model, unless told otherwise)
  3 = channel axis:
        channel 0: raw activation vector at (token t, layer l) — shape (D,)
        channel 1: elementwise delta to the PREVIOUS layer: 
                    raw[l] - raw[l-1]  (elementwise, shape (D,))
        channel 2: elementwise delta to the NEXT layer:
                    raw[l] - raw[l+1]  (elementwise, shape (D,))

Build this tensor independently for Q, K, and V (three separate tensors of 
shape (T, L, D, 3), not stacked together).

IMPORTANT CLARIFICATION NEEDED FROM ME BEFORE YOU IMPLEMENT CHANNELS 1 AND 2:
I described channels 1/2 as "Wasserstein distance per dimension between layer 
L and L±1." For a single scalar value (one dimension, one token, one layer), 
1D Wasserstein distance is mathematically identical to absolute difference 
|a - b|, so as specified this reduces to an elementwise delta (not signed — 
abs difference). Please implement it as SIGNED elementwise delta (raw[l] - 
raw[l-1], not absolute value) since sign carries directional information 
(increasing vs decreasing) that abs() would discard, UNLESS I confirm I 
actually want true distributional Wasserstein distance computed over some 
window/group (e.g. treating a local window of dimensions, or a window of 
tokens, as an empirical distribution and comparing those distributions with 
scipy.stats.wasserstein_distance). Flag this decision back to me rather than 
silently picking one.

EDGE CASES — FIRST AND LALayer 0 has no "previous" layer (l-1) and layer L-1 (0-indexed last layer) 
ST LAYER
has no "next" layer (l+1). Handle this via CIRCULAR WRAP-AROUND indexing:
  - At l=0: "previous" layer wraps to l=L-1 (the last layer)
  - At l=L-1: "next" layer wraps to l=0 (the first layer, i.e. the embedding 
    output / layer 0 hidden state)

Implement this with modulo indexing, e.g. prev_idx = (l - 1) % L, 
next_idx = (l + 1) % L, rather than special-casing the boundaries with if/else.

Please flag explicitly in a code comment that this wrap-around is a design 
choice (not a mathematical necessity) — semantically, wrapping the last 
layer's "next" delta around to the embedding layer conflates "how does this 
representation change with more processing" with "how far has it drifted 
from the raw input embedding," which are conceptually different quantities. 
If this seems off once implemented, alternatives to flag back to me: 
(a) zero-pad the missing boundary delta, (b) duplicate the nearest valid 
delta (edge replication), (c) drop channels 1/2 entirely at the boundary 
layers and let the model handle variable channel availability.
